# Техническое задание: Water Tracker

## 1. Цель проекта
Создание веб-сервиса для персонального расчёта нормы потребления воды с учётом веса, возраста, физической активности и погодных условий. Сервис помогает пользователям следить за гидратацией и достигать дневных целей.

## 2. Роли пользователей
- **Гость:** Может зарегистрироваться.
- **Авторизованный пользователь:** Заполняет профиль (вес, возраст, город, активность), добавляет выпитую воду, видит прогресс и статистику.
- **Администратор:** Управляет пользователями через Django Admin.

## 3. Модели данных (3+ кастомные)

### Модель 1: UserProfile (Профиль пользователя)
- user (OneToOneField → User) — связь с пользователем Django
- weight (FloatField) — вес в кг
- age (IntegerField) — возраст
- city (CharField) — город для получения погоды
- activity_level (CharField с choices) — уровень активности

### Модель 2: DailyNorm (Дневная норма)
- user (ForeignKey → User) — пользователь
- date (DateField) — дата
- calculated_norm (FloatField) — рассчитанная норма воды в мл
- consumed_total (FloatField) — сколько выпито за день
- weather_temp (FloatField, null=True) — температура в этот день

### Модель 3: WaterIntake (Запись о приёме воды)
- user (ForeignKey → User) — пользователь
- amount (IntegerField) — количество воды в мл
- timestamp (DateTimeField) — время добавления

### Связи между моделями
- User → UserProfile: один к одному (OneToOneField)
- User → DailyNorm: один ко многим (ForeignKey)
- User → WaterIntake: один ко многим (ForeignKey)

## 4. Ключевой функционал (User Stories)
1. Пользователь регистрируется на сайте.
2. Пользователь заполняет профиль (вес, возраст, город, активность).
3. Система автоматически рассчитывает норму воды по формуле: вес × 30 мл + коррекция на возраст + коррекция на активность + коррекция на погоду (если температура > 20°C).
4. Пользователь добавляет выпитую воду (250, 500 или 1000 мл).
5. Система отображает прогресс в виде процентного заполнения и показывает, сколько осталось выпить.
6. Пользователь видит погоду в своём городе (интеграция с OpenWeatherMap).

## 5. Внешние интеграции и аналитика
- **API:** OpenWeatherMap — получение текущей температуры по названию города.
- **Аналитика:** Расчёт нормы воды по формуле с учётом погодной коррекции. В будущем планируется добавление графиков прогресса с использованием Matplotlib.
